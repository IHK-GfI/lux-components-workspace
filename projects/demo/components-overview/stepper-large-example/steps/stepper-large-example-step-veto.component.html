Dieser Tab soll zeigen, dass für jeden Schritt eine Veto-Funktion hinterlegt werden kann, die aufgerufen wird, bevor zum nächsten Schritt navigiert wird. Wenn keine Veto-Funktion angegeben wird, hängt das Navigieren vom luxCompleted-Property ab. Das luxCompleted-Property ist bei diesem speziellen Veto-Schritt aktiv. Hier in der Demo wird ein Dialog geöffnet, welcher bestimmt, ob weiter navigiert wird oder nicht. Normalerweise würde die Veto-Funktion übergreifende Validierungen durchführen oder Anfragen an ein Backend (z.B. Berechtigung, BPM,..) stellen.
